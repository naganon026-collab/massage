# SaaS化の構想

多くのお店に使ってもらう場合、「マルチテナントSaaS」としての設計が必要です。

1. **認証機能 (Firebase Auth 等)**
   - アカウント登録・ログイン機能の実装
   - お店ごとのデータを完全に分離するセキュリティ確保

2. **データベースでの設定保存 (Firestore / Supabase 等)**
   - 現在ブラウザ(localStorage)に保存している店舗情報やAPIキーをデータベースに保存
   - どこのPC・スマホからログインしても同じ設定が呼び出せる環境

3. **課金システム (Stripe 等)**
   - 無料トライアル（〇〇回まで生成無料）の導入
   - 月額サブスクリプションで無制限利用などのプラン分け

4. **APIキーの一元管理（または個別入力）**
   - 運営側でOpenAI/GeminiのAPIキーを一括管理するか（ユーザーは手間なし）、
   - ユーザー側で各自のAPIキーを入力させるか（運営側のコストリスクなし）のモデルの決定

5. **WordPress連携用プラグイン開発（応用の方向性）**
   - パスワードを入力させるハードルを下げるため、専用のWordPressプラグインを作ってキーを発行させる仕組みなど
