# パターン別「毎回同じ内容になる」原因と対応

## 調査結果

### なぜヘアカットで「髪が重い」が必ず出るか

1. **タグの固定文言がそのまま渡っている**  
   パターンA（ビフォーアフター）で「ヘアカット」を選ぶと、`TREATMENT_TAGS` の **concern** が毎回そのまま API に送られています。
   - ヘアカットの concern: **「なんとなく重い・スタイルがきまらない」**（`src/types/index.ts`）
   - この文字列が「お客様のお悩み」としてプロンプトに毎回入るため、AI が「髪が重い」系の表現を出しやすい。

2. **任意入力の優先が弱い**  
   「お客様の反応」「ビフォーアフターの変化」を入力しても、プロンプト上では「お悩み」が先にあり、モデルがタグの「重い・スタイルがきまらない」を強く参照していました。任意で書いた内容が従になりがちでした。

3. **お悩みを上書きする入力がなかった**  
   任意項目は「反応」と「変化」のみで、「お悩み」そのものを別の表現に差し替える欄がありませんでした。

### 他のパターンでも「毎回同じ」になりうるか

**はい。** 同じ構造です。

- **パターンB（教育）**: 選んだテーマタグの `theme` / `items` / `solution` がそのまま毎回渡る。同じタグなら同じ入力 → 出力も似る。
- **パターンC/D/E/H**: 同様に、選んだタグの文言がそのまま `additionalContext` に入り、同じタグなら同じプロンプトになる。

「毎回同じ内容」の主因は、**タグの固定文言が毎回そのまま使われ、ユーザーが変えられる部分（任意メモ）が弱い／少ない**ことです。

---

## 実施した対応

### 1. パターンA：任意入力を最優先にする（プロンプト修正）

- 「お客様の反応」または「ビフォーアフターの具体的な変化」が**入力されているとき**は、それらを投稿の中心にし、フック・本文の「お悩み」は**任意入力に合わせる**よう指示を追加。
- 任意入力に「重い」「パサつき」等が含まれていなければ、それらは使わない旨を明示。
- フックのルールを「任意入力があるときは、任意入力の内容に合った悩みで始め、タグの文言をそのまま使わない」に変更。

### 2. パターンA：「お悩み（任意）」欄の追加

- ビフォーアフターの「より具体的な投稿にする（任意）」内に **「お悩み（任意）」** を追加。
- ここに文字がある場合は、**タグの concern の代わりに**この値を API に送るように変更（`additionalContext.concern` を上書き）。
- 例：「前髪が伸びて目にかかる」「くせ毛で広がる」「ボブにしたいけど似合うか不安」など、ヘアカットでも「重い」以外の悩みを書ける。

これにより、

- お悩みを指定したいとき → 「お悩み（任意）」に書く  
- 反応・変化を指定したいとき → 従来どおり「反応」「ビフォーアフターの変化」に書く  

の両方で、タグの「なんとなく重い・スタイルがきまらない」に引っ張られにくくなります。

---

## 今後の検討（他パターン）

- **パターンB**: 教育テーマタグに加え、「今回の切り口（任意）」のような自由入力欄を設け、あればタグの theme を補足・上書きする。
- **パターンD/E/H**: 同様に、タグで渡す文言に加えて「自由メモ」を強く効かせるプロンプトにする、または「〇〇（任意）」欄を増やして上書きできるようにする。

現時点では、**パターンA（ビフォーアフター）** について上記の対応を実施済みです。
