# LogicPost 価格設定戦略（APIコスト考慮）

## コスト要因の整理

### 高コストのAPI

| 機能 | API | コスト要因 |
|------|-----|-----------|
| **ブログ生成** | `generate-ortho-blog` | 長文HTML生成、トークン消費が大きい（数千〜数万トークン） |
| **メイン生成** | `generate` | 5媒体分を一括生成、プロンプト長い。portal（ブログ用）含むとさらに増 |
| **画像からの生成** | `generate-image` | 画像分析＋5媒体生成、マルチモーダルで高コスト |
| **LINE投稿** | `post-to-line` | LINE API 自体は無料だが、ユーザーのLINE公式アカウント料金に影響 |
| **Instagram・GBP投稿** | `post-to-late` | Late API の従量課金（LogicPost側のLate契約コスト） |

### コストの重さ（イメージ）

1. **ブログ（portal）生成** … 最も重い（長文・HTML・複数セクション）
2. **画像からの生成** … 重い（画像分析＋生成）
3. **通常生成（portal含む）** … 中〜重（5媒体一括）
4. **Late投稿** … 1回ごとにLate APIコスト
5. **LINE投稿** … LogicPost側の直接コストは小（ユーザーがLINEに支払い）

## 価格設定の方向性

### 方針A：機能別プラン（推奨）

コストの高い機能を有料プランで区切り、使う機能に応じて価格を変える。

| プラン | 月額 | 生成回数 | 含まれる機能 | 想定コストカバー |
|--------|------|----------|-------------|-----------------|
| **ライト** | 980円 | 月30回まで | 生成のみ（Instagram・GBP・LINE用テキスト）。ブログ・ショート動画なし。投稿機能なし | 生成APIコストのみ |
| **スタンダード** | 2,480円 | 月100回まで | 上記＋ブログ生成＋ショート動画台本。投稿機能（LINE・Late）あり | 生成＋投稿の標準利用 |
| **プロ** | 3,980円 | 無制限 | 全機能＋画像からの生成。投稿回数無制限 | ヘビーユーザー向け |

### 方針B：使用量ベース

| プラン | 月額 | 生成回数 | ブログ生成 | 投稿回数 |
|--------|------|----------|-----------|----------|
| **無料** | 0円 | 5回/月 | 0回 | 0回 |
| **ベーシック** | 1,480円 | 30回/月 | 3回/月 | 10回/月 |
| **スタンダード** | 2,980円 | 100回/月 | 10回/月 | 50回/月 |
| **アンリミテッド** | 4,980円 | 無制限 | 20回/月 | 無制限 |

※ ブログ・投稿を別枠にすることで、高コスト機能の濫用を防ぐ

### 方針C：現行維持＋オプション課金

- **基本プラン**: 2,980円（現状のアンリミテッド）
- **ブログ生成**: 1回あたり +50円 または 月5回まで無料、以降 +30円/回
- **投稿機能**: 月20回まで無料、以降 +10円/回

## 推奨：方針A をベースにした段階的導入

### 第1段階（まず導入しやすい形）

| プラン | 月額 | 生成回数 | 内容 |
|--------|------|----------|------|
| **無料** | 0円 | 月5回まで | 投稿不可 |
| **スタンダード** | 2,480円 | 月100回まで | 投稿可（LINE・Late）、ブログ・ショート動画含む |

→ 現行2,980円を2,480円に下げつつ、無料プランから投稿を外すことで「投稿＝有料」を明確に。

### 第2段階（需要が見えたら）

| プラン | 月額 | 生成回数 | 内容 |
|--------|------|----------|------|
| **無料** | 0円 | 月5回まで | 投稿不可 |
| **ライト** | 980円 | 月30回まで | ブログ・ショート動画なし、投稿不可 |
| **スタンダード** | 2,480円 | 月100回まで | 投稿可、ブログ・ショート動画含む |
| **プロ** | 3,980円 | 無制限 | 全機能、画像からの生成、優先サポート等 |

## 実装時のポイント

1. **ブログ生成の制限**  
   `generate-ortho-blog` をプランごとに回数制限（例：スタンダード月10回、プロ月30回）

2. **投稿回数の制限**  
   `post-to-line` / `post-to-late` の呼び出し回数をプラン別にカウント・制限

3. **Stripe の複数 Price**  
   各プラン用の Price を Stripe で作成し、checkout 時に選択

4. **outputTargets との連動**  
   設定の「出力する媒体」で portal（ブログ）をオフにしているユーザーは、ブログ生成コストがかからない前提でプラン設計可能
